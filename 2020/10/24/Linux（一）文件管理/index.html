Linux目录结构

/bin 可执行的命令文件 /root 根用户的家目录
/boot 系统核心文件和开机所需文件 /sbin root用户才能执行的命令文件
/dev 系统设备相关文件 /srv 服务启动后访问的数据文件
/etc 系统配置文件 /tmp 程序临时存放的文件
/home 非root用户的家目录 /opt 第三方软件安装目录
/lib 系统和程序的库函数文件 /media 移动设备相关文件

目录操作命令

  • cd

    命令格式：cd [ 目录名称 ]

    功能描述：将当前目录切换到指定的目录；若无指定目录，则回到用户主目录。

    执行权限：All User

  • pwd

    命令格式：pwd [ 选项 ]

    功能描述：显示用户当前所处的工作目录

    执行权限：All User

  • mkdir

    命令格式：mkdir 选项 [ -p ] 目录名

    功能描述：创建目录

    ​ • -p 表示创建嵌套目录

    ​ • -v 显示目录创建信息

    执行权限：All User

  • rmdir

    命令格式：rmdir 选项 [ -p ] 目录名

    功能描述：删除空目录

    ​ • -p 可以递归的删除空目录

    执行权限：All User

    ​ • 创建重复的文件会修改时间戳

文件属性

  • 列出文件和属性命令 ls

    命令格式：ls [ 选项 ] [ 文件 ]

    功能描述：显示指定路径下的目录、文件列表

    ​ • -a 显示所有文件，包括隐藏文件

    ​ • -l 使用长格式显示文件详细信息

    ​ • -i 查看文件节点号

    执行权限：All User

  • 权限修改 chmod

    修改文件权限：chmod [ 权限 ] [ 文件 ]

    示例：chmod 744 test.sh

    修改所属用户和用户组：chown [ 用户 ] : [ 用户组 ] [ 文件名 ]

    示例：chown mandy:root test.sh

查看文件命令

  • 查看文件 cat

    命令格式：cat [ 选项 ] [ 文件 ]

    功能描述：查看较小的文本文件

    ​ • -b 显示文件每行的行编号（不包括空行）

    ​ • -E 在每行的未尾显示$符号，表示行结束

    ​ • -n 显示文件所有行的行编号

    ​ • -s 将相连的多个空行用一个空行来代替

    示例图：

  • 连接合并文件 cat

    命令格式：cat file1 file2 > file3

    格式解释：>为重定向符，file3可以为不存在的文件

    示例图：

  • 查看文件 more

    命令格式：more [ 选项 ] [ 文件 ]

    功能描述：查看较大的文本文件

    ​ • -number 指定一个整数 ，表示一屏显示多少行

    ​ • -d 在屏幕底部显示帮助信息

    ​ • -s 将相连的多个空行用一个空行来代替

    进入more状态后：

    ​ • q 退出

    ​ • Enter键 显示下一行

    ​ • 空格键 显示下一页

  • 查看文件 less

    命令格式：less [ 选项 ] [ 文件 ]

    功能描述：常用于显示较大文件的内容

    ​ • -M 在显示屏的最后一行输出：文件名、当前显示的行数、总行数和光标所处文档位置

    ​ • -s 将相连的多个空行用一个空行来代替

    ​ • -S 截断过长的行而不换行显示

    less状态的上下查看功能：

    ​ 对于较大的文件，一屏显示不完，使用上下箭头可以翻看文件的内容

    less状态的查找功能：

    ​ 在文件底部输入：/关键字

    ​ 输入的关键字将被高亮显示

  • 查看文件 head/tail

    命令格式： head/tail [ 选项 ] [ 文件 ]

    功能描述：显示文件头/尾的内容，默认10行，常常用于服务器维护的查看日志场景

    ​ • -K 显示文件的前/后K行内容

    ​ • -c K显示每个文件的前K字节内容

    ​ • -f 实时刷新文件(用于tail命令)

通配符

​

***** 代表任意长度字符串（包括0长度）
? 代表任意单个字符
[c1-c2] 匹配c1c2字符序列中的任意的单一字符（前提：c1c2是一个有序的字符序列，一般为数字序列或英文字符序列）
[c1c2…cn] [c1,c2,…,cn] 匹配中括号之间字符列表中的任意的单一字符
{string1,string2,…,stringN} 表示匹配string1 string2 直至stringN这N个字符串中的某个字符串
! 感叹号往往和[]与{}联用，表示对其中的内容取反

查找文件命令

  • 文件查找 find

    功能： 在指定的目录结构中搜索文件，并执行指定的操作。

    用法： find [ 查找范围 ] <查找条件> [ 动作 ]

    查找范围： 指定要查找的目录 ，省略表示在当前目录及子目录中查找。

    查找条件： 指定查找文件的具体条件，如：文件名、类型、大小、修改时间等。

    动 作： 对查找的文件做的进一步处理 。

  • 根据文件名查找

    find [ 查范围 ] -name 文件名 根据文件名查找

    find [ 查找范围 ] -iname 文件名 据文件名查找，忽略大小写。

    文件名：

    • 精确的名称

    • 含有通配符的关键字，关键字需要加引号“”或 ’’

    示例图：

  • 根据文件类型查找

    find [ 查找范围 ] –type 文件类型字符

    示例图：

  • 根据时间属性查找

    find [ 查找范围 ] -mtime n 查找过去n天被修改过的文件

    -mmin n 在过去 n 分钟内被修改过
    -atime n 在过去n天内被读取/root/tmp执行过的文件
    -amin n 在过去 n 分钟内被读取/root/tmp执行过的文件
    -ctime n 在过去n 天内属性被修改过的文件
    -cmin n 在过去n 分钟内属性被修改过的文件

    时间范围：

  • find的复合条件

    逻辑运算符 - a –o ! 组成复合条件

    ​ • -a and 表示 “与”

    ​ • -o or 表示 “或”

    ​ • -! not 表示 “取反”

  • 其他查找条件

    -size -n c/k/m/g 根据文件大小(n/nk/nm/ng字节)查找
    -perm 文件权限数字 根据文件的权限进行查找
    -user ‘用户名’ 根据用户名查找
    -uid 用户id 根据用户id查找
    -group ‘用户组名‘ 根据用户组名查找
    -gid 用户id 根据用户组id查找

  • find命令查到结果后对文件的操作

    命令格式：find <-option> [-print | -ls] [-exec|-ok ;]
